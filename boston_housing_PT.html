Implementação: Definir uma Métrica de Desempenho¶

É difícil medir a qualidade de um modelo dado sem quantificar seu desempenho durante o treinamento e teste. Isso é geralmente feito utilizando algum tipo de métrica de desempenho, através do cálculo de algum tipo de erro, qualidade de ajuste, ou qualquer outra medida útil. Para este projeto, você irá calcular o coeficiente de determinação, R2, para quantificar o desempenho do seu modelo. O coeficiente de determinação é uma estatística útil no campo de análise de regressão uma vez que descreve o quão "bom" é a capacidade do modelo em fazer estimativas.

Os valores para R2 têm um alcance de 0 a 1, que captura a porcentagem da correlação ao quadrado entre a estimativa e o valor atual da variável alvo. Um modelo R2 de valor 0 sempre falha ao estimar a variável alvo, enquanto que um modelo R2 de valor 1, estima perfeitamente a variável alvo. Qualquer valor entre 0 e 1 indica qual a porcentagem da variável alvo (ao utilizar o modelo) que pode ser explicada pelos atributos. Um modelo pode dar também um R2 negativo, que indica que o modelo não é melhor do que aquele que estima ingenuamente a média da variável alvo.

Questão 7 - Busca em Matriz¶

  • O que é a técnica de busca em matriz (grid search)?
  • Como ela pode ser aplicada para otimizar um algoritmo de aprendizagem?

Dica: Quando explicar a técnica de busca em matriz, tenha certeza que você explicou o motivo dela ser usada, o que a 'matriz' significa nesse caso e qual o objetivo da técnica. Para ter uma resposta mais sólida, você pode também dar exemplo de um parâmetro em um modelo que pode ser otimizado usando essa técnica.

Resposta: Em geral, para decidir sobre o modelo que melhor generaliza os dados, precisamos:

  1. Dividir os dados entre os grupos de treinamento, validação e teste
  2. A partir do modelo escolhido, definimos os hyper-parâmetros e parâmetros que definem o modelo com melhor generaliação.

Para decidir os valores do modelo de melhor generalização, mantemos o hyper-parâmetro fixo - grau do polinômio, profundidade da árvore de decisão - e variamos os parâmetros - coeficientes do polinômio, nós da árvore - avaliando o erro do modelo. Contudo, em alguns casos, temos modelos que tem mais de um hyperparâmetro e, nesse cenário, que a técnica de busca em matriz - ou busca em grade - se faz importante.

Na busca em matriz, construímos de fato uma matriz com os hyper-parâmetros possíveis do modelo. Para cada coluna ou linha, um desses hyper-parâmetros é mantido fixo e o outro varia em alguns valores possíveis. Para cada combinação da matriz, avaliamos o erro do modelo para escolher qual melhor generaliza os dados.

Como exemplo, podemos considerar uma implementação em que o modelo escolhido será o de Máquina de Suporte a Vetor (SVM). Nesse tipo de modelo, temos dois hyper-parâmetros. O kernel, que define o formato da área de separação dos dados, e o gamma, que define as fronteiras da área. Sendo assim, precisamos utilizar a busca em matriz para encontrar o melhor modelo. Deixaremos um valor de kernel fixo e avaliamos o modelo para diferentes valores de gamma. Depois, alteramos o valor do kernel e testamos o modelo para os mesmos valores de gamma já testado. A combinação com erro menor - ou pontuação maior - é o melhor modelo para nosso conjunto de dados.

  • O que é a técnica de treinamento de validação-cruzada k-fold?
  • Quais benefícios essa técnica proporciona para busca em matriz ao otimizar um modelo?

Dica: Lembre-se de expllicar o que significa o 'k' da validação-cruzada k-fold, como a base de dados é dividida e quantas vezes ela é executada.

Assim como há um raciocínio por trás de utilizar um conjunto de teste, o que poderia dar errado ao utilizar busca em matriz sem um conjunto de validação cruzada? Você pode utilizar a documentação para basear sua resposta.

Resposta: Quando vamos avaliar a performance de um modelo, precisamos treinar o modelo com o conjunto de treinamento e depois testar se o modelo consegue generalizar os dados com o conjunto de teste.

Entretanto, algumas técnicas de seleção do modelo - a exemplo da busca em matriz - consideram testar o modelo antes de selecioná-lo para avaliar qual teria a melhor performance em um grupo de teste.

Para esse fim, uma opção seria avaliar o modelo com o próprio grupo de teste, porém esse conjunto de dados não deve ser utilizado até que a intenção seja realmente testar o modelo escolhido e, um outro problema, é que corremos o risco de acabar com um modelo overfitted, pois o modelo acaba aprendendo com o conjunto de teste.

Uma soluçao é criar um terceiro conjunto de dados chamado de "conjunto de validação". Esse conjunto será utilizado como se fosse o conjunto de teste para validar qual o melhor modelo e parâmetros para a melhor generalização. Contundo, acabamos perdendo mais alguns pontos que poderiam ser utilizados para treinar nosso modelo. Para evitar a "perda de dados" existe a validação cruzada k-fold.

Nessa técnica, separamos nossos dados nos três conjuntos necessários para treinamento, validação e teste. Contudo, fazemos isso considerando "k" grupos distintos de dados em que, em cada grupo, teremos os conjuntos de treinamento, validação e teste diferentes. Quando construímos o modelo, o teste é executado em todos esses "k" grupos para avaliar a melhor resposta aos dados.

Nesse caso, não estamos perdendo nenhum dado, pois todos são utilizados para treinamento, validação e teste, embora esses conjuntos sejam diferentes a cada iteração dos testes. Por utilizar todos os dados para as três finalidades que essa técnica é considerada como se fizesse a reciclagem dos dados.

def fit_model(X, y):

    # Gerar conjuntos de validação-cruzada para o treinamento de dados
