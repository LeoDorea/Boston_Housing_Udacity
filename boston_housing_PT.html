Implementação: Definir uma Métrica de Desempenho¶

difcil medir a qualidade de um modelo dado sem quantificar seu desempenho durante o treinamento e teste. Isso geralmente feito utilizando algum tipo de mtrica de desempenho, atravs do clculo de algum tipo de erro, qualidade de ajuste, ou qualquer outra medida til. Para este projeto, voc ir calcular o coeficiente de determinao, R2, para quantificar o desempenho do seu modelo. O coeficiente de determinao uma estatstica til no campo de anlise de regresso uma vez que descreve o quo "bom" a capacidade do modelo em fazer estimativas.

Os valores para R2 tm um alcance de 0 a 1, que captura a porcentagem da correlao ao quadrado entre a estimativa e o valor atual da varivel alvo. Um modelo R2 de valor 0 sempre falha ao estimar a varivel alvo, enquanto que um modelo R2 de valor 1, estima perfeitamente a varivel alvo. Qualquer valor entre 0 e 1 indica qual a porcentagem da varivel alvo (ao utilizar o modelo) que pode ser explicada pelos atributos. Um modelo pode dar tambm um R2 negativo, que indica que o modelo no melhor do que aquele que estima ingenuamente a mdia da varivel alvo.

Questão 7 - Busca em Matriz¶

  • O que a tcnica de busca em matriz (grid search)?
  • Como ela pode ser aplicada para otimizar um algoritmo de aprendizagem?

Dica: Quando explicar a tcnica de busca em matriz, tenha certeza que voc explicou o motivo dela ser usada, o que a 'matriz' significa nesse caso e qual o objetivo da tcnica. Para ter uma resposta mais slida, voc pode tambm dar exemplo de um parmetro em um modelo que pode ser otimizado usando essa tcnica.

Resposta: Em geral, para decidir sobre o modelo que melhor generaliza os dados, precisamos:

  1. Dividir os dados entre os grupos de treinamento, validao e teste
  2. A partir do modelo escolhido, definimos os hyper-parmetros e parmetros que definem o modelo com melhor generaliao.

Para decidir os valores do modelo de melhor generalizao, mantemos o hyper-parmetro fixo - grau do polinmio, profundidade da rvore de deciso - e variamos os parmetros - coeficientes do polinmio, ns da rvore - avaliando o erro do modelo. Contudo, em alguns casos, temos modelos que tem mais de um hyperparmetro e, nesse cenrio, que a tcnica de busca em matriz - ou busca em grade - se faz importante.

Na busca em matriz, construmos de fato uma matriz com os hyper-parmetros possveis do modelo. Para cada coluna ou linha, um desses hyper-parmetros mantido fixo e o outro varia em alguns valores possveis. Para cada combinao da matriz, avaliamos o erro do modelo para escolher qual melhor generaliza os dados.

Como exemplo, podemos considerar uma implementao em que o modelo escolhido ser o de Mquina de Suporte a Vetor (SVM). Nesse tipo de modelo, temos dois hyper-parmetros. O kernel, que define o formato da rea de separao dos dados, e o gamma, que define as fronteiras da rea. Sendo assim, precisamos utilizar a busca em matriz para encontrar o melhor modelo. Deixaremos um valor de kernel fixo e avaliamos o modelo para diferentes valores de gamma. Depois, alteramos o valor do kernel e testamos o modelo para os mesmos valores de gamma j testado. A combinao com erro menor - ou pontuao maior - o melhor modelo para nosso conjunto de dados.

Entretanto, algumas tcnicas de seleo do modelo - a exemplo da busca em matriz - consideram testar o modelo antes de selecion-lo para avaliar qual teria a melhor performance em um grupo de teste.

Para esse fim, uma opo seria avaliar o modelo com o prprio grupo de teste, porm esse conjunto de dados no deve ser utilizado at que a inteno seja realmente testar o modelo escolhido e, um outro problema, que corremos o risco de acabar com um modelo overfitted, pois o modelo acaba aprendendo com o conjunto de teste.

Uma soluao criar um terceiro conjunto de dados chamado de "conjunto de validao". Esse conjunto ser utilizado como se fosse o conjunto de teste para validar qual o melhor modelo e parmetros para a melhor generalizao. Contundo, acabamos perdendo mais alguns pontos que poderiam ser utilizados para treinar nosso modelo. Para evitar a "perda de dados" existe a validao cruzada k-fold.

Nessa tcnica, separamos nossos dados nos trs conjuntos necessrios para treinamento, validao e teste. Contudo, fazemos isso considerando "k" grupos distintos de dados em que, em cada grupo, teremos os conjuntos de treinamento, validao e teste diferentes. Quando construmos o modelo, o teste executado em todos esses "k" grupos para avaliar a melhor resposta aos dados.

Nesse caso, no estamos perdendo nenhum dado, pois todos so utilizados para treinamento, validao e teste, embora esses conjuntos sejam diferentes a cada iterao dos testes. Por utilizar todos os dados para as trs finalidades que essa tcnica considerada como se fizesse a reciclagem dos dados.

    # Gerar conjuntos de validao-cruzada para o treinamento de dados
